# Triage labels

Not applicable. This repo has no issue tracker (see
`docs/agents/issue-tracker.md`), so the `triage` state machine and its labels
are unused.

If an issue tracker is configured later, the five canonical triage roles and
their label strings would be defined here:

| Role              | Meaning                                          |
|-------------------|--------------------------------------------------|
| `needs-triage`    | maintainer needs to evaluate                     |
| `needs-info`      | waiting on reporter                              |
| `ready-for-agent` | fully specified, AFK-ready                       |
| `ready-for-human` | needs human implementation                       |
| `wontfix`         | will not be actioned                             |
